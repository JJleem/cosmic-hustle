---
name: 유저 작업 방식·선호도
description: 작업 스타일, 선호하는 접근법, 피해야 할 것들
type: feedback
---

- 탐색적 질문("어떻게 생각해?")엔 2~3문장 추천 + 주요 트레이드오프만.
  구현은 동의 받은 후 시작.
  **Why:** 먼저 구현하면 방향이 틀렸을 때 낭비.

- 커밋·푸시는 명시적으로 요청할 때만.
  **Why:** 유저가 직접 타이밍 컨트롤 원함.

- 레퍼런스 이미지 있는 이미지 생성 프롬프트는 외형 묘사 제거,
  동작·포즈만 기술. 단, 레퍼런스 없을 때는 디테일 묘사 포함.
  **Why:** 외형 텍스트 묘사가 레퍼런스를 오버라이드해서 캐릭터 디자인 붕괴.
  **반례:** 첫 새벽마감 프롬프트(디테일 묘사 있는 버전)가 오히려 잘 나옴.
  레퍼런스 3장 제공 시에만 묘사 제거.

- 한국어로 대화. 코드 주석·커밋 메시지도 한국어 혼용 OK.
