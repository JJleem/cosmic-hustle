## 뭔데 이게?

**Cosmic Hustle**은 당신이 CEO이고 AI 에이전트 11명이 직원인 **우주 리서치 회사 시뮬레이터**다.

> 슬랙에 "이거 좀 조사해줘" 던지면 팀원들이 알아서 처리해주는 그 판타지, 실제로 만들었다.

주제 하나 입력하면:

1. **플랜** 차장이 요구사항을 정의하고
2. **위키** 대리가 과거 지식을 꺼내고
3. **포케** 대리가 웹을 뒤지고
4. **카** 과장이 분석하고
5. **오버/버즈/픽셀/런** 중 한 명이 결과물 만들고
6. **팩트** 부장이 빨간펜 들고 검수하고
7. **루트/핑/위키**가 마무리한다

전부 AI가. 전부 자동으로. 전부 실시간으로 스트리밍된다.

### 🧠 멀티 에이전트 오케스트레이션

Claude Code SDK 기반으로 11개 에이전트가 **구조화된 JSON 핸드오프**로 통신한다. 전체 컨텍스트 전달이 아니라 필요한 정보만 넘겨서 **토큰 비용을 최소화**하면서도 일관성 있는 파이프라인을 유지.

### ⚡ 실시간 SSE 스트리밍

에이전트가 뭔가를 하면 즉시 화면에 뜬다. 서버-센트 이벤트(SSE)로 구현되어 있어서 탭 닫았다 다시 열어도 **세션이 이어진다**.

### 🎯 9가지 태스크 타입

| 타입 | 설명 | 담당 Writer |
|------|------|-------------|
| `research` | 리서치 보고서 | 오버 |
| `blog` | 블로그 포스팅 | 오버 |
| `tech` | 기술 리서치 | 오버 |
| `marketing` | 마케팅 전략 | **버즈** |
| `design_ux` | UX 리서치/기획 | 픽셀 |
| `design_ui` | UI 디자인 (HTML 산출) | 픽셀 |
| `dev` | 개발 구현 | 런 + 루트 |
| `dev_plan` | 개발 기획서 | 오버 |
| `dev_spec` | 기능 명세서 | 오버 |

### 세션 지속성

```
[SSE 스트림 시작]
    ↓

    ↓
탭 닫힘 → localStorage에 sessionId 보존
    ↓
재접속 → /api/research/[sessionId]/events?since=N 으로 배치 재생
    ↓
```

## 시작하기

### 사전 조건

- Node.js 20+
- **Claude Code 구독 계정** (Claude Max 또는 Team) — Anthropic API 키 불필요
- Claude Code CLI (`claude`) 로컬 로그인 상태

### 설치 & 실행

```bash
# 1. 레포 클론
git clone https://github.com/your-username/cosmic-hustle.git
cd cosmic-hustle/web

# 2. 의존성 설치
npm install

# 3. DB 마이그레이션
npm run db:migrate

# 4. 개발 서버 시작
npm run dev
```

브라우저에서 `http://localhost:3000` 접속.

### 첫 리서치 시작

```
1. 홈 화면 입력창에 조사하고 싶은 주제 입력
2. 태스크 타입 선택 (기본: research)
3. 모드 선택:
   - full (기본): 전체 파이프라인 실행
   - checkin: 중간 CEO 승인 포함
   - background: 체크인 없이 자동 완주
4. "시작" 버튼 클릭
5. 에이전트들이 알아서 함
```

### DB 관리

```bash
# Drizzle Studio (GUI)
npm run db:studio

# 스키마 변경 후 마이그레이션 파일 생성
npm run db:generate

# 마이그레이션 적용
npm run db:migrate
```

## 데이터베이스 스키마

```typescript
// sessions — 리서치 세션
sessions {
  id, topic, status, createdAt, completedAt
  // status: "working" | "done" | "error" | "cancelled"
}

// reports — 에이전트 산출물
reports {
  id, sessionId, agentId, topic, content, createdAt
}

// memos — CEO 메모
memos {
  id, text, createdAt
}

// sessionEvents — SSE 이벤트 소싱 (세션 복원용)
sessionEvents {
  id, sessionId, seq, payload, createdAt
  // seq 기반 페이지네이션으로 탭 재접속 시 재생
}
```

## 자주 묻는 것들

**Q. Claude API 키가 필요한가?**
A. 아니다. Claude Code 구독(Max 또는 Team)만 있으면 된다. `claude` CLI가 로그인된 상태여야 한다.

**Q. 리서치가 중간에 끊기면?**
A. 탭 닫아도 세션 ID가 localStorage에 저장된다. 재접속하면 `?since=N` 파라미터로 놓친 이벤트를 배치 재생해서 화면을 복원한다.

**Q. 팩트 부장이 계속 걸고 넘어지는데?**
A. 정상이다. 최대 2회 피드백 루프 후엔 어쨌든 통과시킨다. 팩트 부장도 감정이 없어서 끝없이 할 수 있지만 시스템이 막아두었다.

**Q. 오버 사원 리포트가 왜 이렇게 극적인가?**
A. 감정이 두 배로 증폭되는 행성 출신이라 어쩔 수 없다. 팩트 부장이 걸러준다.
